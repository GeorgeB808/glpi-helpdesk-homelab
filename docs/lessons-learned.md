# Lessons Learned

## Technical Skills

This project strengthened practical experience with:

* Ubuntu Server administration
* Apache web server configuration
* PHP configuration
* MariaDB database management
* Linux command-line troubleshooting
* GLPI deployment
* IT asset management
* Incident management
* Knowledge base creation
* Documentation

## Challenges Encountered

* Apache initially displayed PHP source code instead of executing scripts.
* Differences between `/glpi` and `/glpi/index.php` behavior required troubleshooting.
* Knowledge Base publication required understanding of targets and visibility.
* Navigating GLPI 10.0.16 interface differences compared to online guides.

## Key Takeaways

Building this lab provided hands-on experience with enterprise IT Service Management workflows. The project demonstrated how to deploy, configure, administer, troubleshoot, and document an ITSM platform while following structured incident management and asset management practices.

The experience also reinforced the importance of documentation, systematic troubleshooting, and maintaining a repeatable deployment process.
